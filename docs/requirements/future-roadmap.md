# Future Feature Roadmap

Ideas captured for later — **not scoped, not designed, not scheduled into a phase.** Nothing here should influence Phase 1 implementation. This list exists so an idea raised today isn't lost, and so future architecture decisions (e.g., a new field, a new event) can be made with an eye on what might land here, without building for it now.

Revisit this list when scoping Phase 2+ or when Phase 1 is stable in production — at that point, promote an item into `functional-requirements.md` with real FR-IDs before building it.
